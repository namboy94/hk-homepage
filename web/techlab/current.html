Running Turbocash on modern hardware

Legacy DOS applications can be a nightmare to run, especially so if they are dependent on the frequency of the CPU. These programs will sometimes experience issues like "divided by zero"-errors on modern hardware, because the CPU frequency is too high for the program to handle correctly.

Turbocash, a DOS-based accounting program is one of these. There are several workarounds required to gain the full functionality of the software on modern operating systems and hardware.

What you'll need:

  • An installation disk for Windows 98 SE or an ISO of it.
  • A valid Windows 98 SE license key
  • DOSBox Megabuild
  • Oracle Virtualbox
  • A copy of Turbocash

To start, set up a new Windows 98 Virtual Machine in Oracle Virtualbox using the Windows 98 ISO. If you only have an installation CD, create an ISO of it beforehand, or use Virtualbox's built-in functionality to use your CD Drive.
Use the following settings for the Virtual machine:

  • 265MB RAM
  • 2GB HDD Space (Use VHD)
  • PIIX3 Chipset
  • IO-APIC activated
  • 128M Graphics RAM
  • 2D and 3D accelearation activated
  • Bridged Network

Now, we'll start installing Windows 98 SE on the virtual machine. Just use the default settings for everything and enable large disk support.
To enable printing over the network, which will be necessary to print statements, you will have to do the following steps:

  1. Open the Control Panel
  2. Go to Network Settings
  3. Add a new service
  4. Select "File and Printer sharing for Microsoft networks"
  5. Confirm the selections
  6. Restart the Virtual Machine

Now, we'll have to install the software CPUKiller. The newest version is 3.0 and requires a license, but older versions of the software work just as well and are as far as I can tell free to use.
This software enables you to slow down the CPU, which will be necessary to run Turbocash.
