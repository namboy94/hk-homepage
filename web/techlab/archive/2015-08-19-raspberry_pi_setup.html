Setting up a Raspberry Pi as an HTML server

Raspberry Pis are perfect for use as a small little HTML server. You shouldn't use it to host Google, but a small little homepage is perfectly fine.

This article will show you how to set up a Raspberry Pi as an HTML server as well as how to set up remote access and Adafruit TFT Monitors.

This article assumes the Pi is running on Raspbian. You need to SSH into the Raspberry Pi or otherwise have access to the Pi's terminal.

Setting up the Apache Webserver:

Enter the following commands:

sudo apt-get install apache2 -y
cd /var/www
sudo chown pi: index.html
Test if the webserver is running by visiting the Pi's IP Adress in a Web Browser.

Setting up VNC for remote control:

Enter the following commands:

sudo apt-get install tightvncserver
tightvncserver

You'll have to enter a password now, make sure it's a strong.
VNCServer is now installed and ready to be used. To start a server you can use the following command:

vncserver :0 -geometry 1920x1080 -depth 24
To start the VNC Server at boot, you'll have to do the following:

cd /etc/init.d/
sudo nano vncboot
Copy the contents of this file to the terminal. Then press CTRL-X and save the file. Once that's done, run the followinmg commands:

sudo chmod 755 vncboot
sudo update-rc.d vncboot defaults
And afterwards reboot the Raspberry Pi.

Setting up Adafruit TFT Monitors:

Enter the following commands:

curl -SLs https://apt.adafruit.com/add | sudo bash
sudo apt-get install raspberrypi-bootloader
sudo apt-get install adafruit-pitft-helper
The next step is dependent on the specific model used.

For the 2.4" Touch Model:

sudo adafruit-pitft-helper -t 28r
For the 2.2" Non-Touch Model:

sudo adafruit-pitft-helper -t 22
Afterwards, reboot the Raspberry Pi.
