Using Rsync for backing up or transferring files

Rsync is a very useful program to backup files or even just for transferring them. It verifies every operation using checksums, making sure that no data loss occurs during the data transfer. This makes it superior to operations like mv.

The basic syntax of an rsync command is as follows:

rsync options source destination
The most important options are:

  • -a Archive Mode, should normally be used
  • -v Verbose Mode, prints the perfrmed actions to the terminal
  • -n Dry Run Mode, for testing. If this option is enabled, no changes will be made
  • --delete-after Deleted files that don't exist in the source directory after the rsync command itself

Using rsync with a remote server, for instance via SSH, you'll need the following syntax:

user@server:path

Use case Examples:

Copying files from one local drive to another:
rsync -av /mnt/disk1/folder/ /mnt/disk2/folder
Backing up the Home drive to a NAS via SSH:
rsync -av /home/user user@server:/mnt/disk/backups
List files missing from your PC that are available on a remote server:
rsync -anv user@server:/mnt/disk/secrets/ /home/user/secrets
Synchronize a folder on a server with a local folder, making the remote folder the same as the local folder:
rsync -anv --delete-after /home/user/stuff/ user@server:/mnt/disk/stuff

Note that sometimes an "/" is appended to the end of a directory. If this is the case, the content of the directory is synced. If the "/" is ommited, The folder itself is synced.
