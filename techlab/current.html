Running Turbocash and other frequency-dependent DOS programs on modern hardware

Legacy DOS applications can be a nightmare to run, especially so if they are dependent on the frequency of the CPU. These programs will sometimes experience issues like "divided by zero"-errors on modern hardware, because the CPU frequency is too high for the program to handle correctly.

Turbocash, a DOS-based accounting program is one of these. There are several workarounds required to gain the full functionality of the software on modern operating systems and hardware.

What you'll need:

  • An installation disk for Windows 98 SE or an ISO of it.
  • A valid Windows 98 SE license key
  • DOSBox Megabuild
  • Virtualbox
  • A copy of Turbocash/the program required to run
